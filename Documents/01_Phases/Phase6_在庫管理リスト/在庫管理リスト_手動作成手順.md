# 在庫管理リスト - 手動作成手順

**対象サイト**: https://cellgentech.sharepoint.com/sites/SP__Prototype  
**リスト名**: 在庫管理

---

## Step 1: リストの作成

1. SharePointサイトを開く
2. ホームページで **「+ 新規」** → **「リスト」** をクリック
3. **「空白のリスト」** を選択
4. 名前に `在庫管理` と入力
5. **「作成」** をクリック

---

## Step 2: 列の追加

### 基本列の設定

まず「タイトル」列の名前を変更します：
- **「タイトル」** 列のヘッダーをクリック → **「列の設定」** → **「名前の変更」**
- `品名` に変更

### 追加する列一覧

以下の列を順番に追加します。  
**追加方法**: 列ヘッダーの右端にある **「+」** → **「列を追加」** または **「さらに表示」**

| # | 表示名 | 種類 | 必須 | 備考 |
|---|--------|------|------|------|
| 1 | メーカー | 1行テキスト | ○ | |
| 2 | メーカーコード | 1行テキスト | ○ | 製品番号 |
| 3 | 枝番 | 1行テキスト | | ロット別管理用(-1,-2) |
| 4 | 包装形状 | 1行テキスト | | |
| 5 | 入数/箱 | 数値 | | |
| 6 | 発注業者 | 1行テキスト | | |
| 7 | 発注ID | 1行テキスト | | YYDD-000-PJ形式 |
| 8 | 単価 | 通貨 | | |
| 9 | 現在在庫数 | 数値 | ○ | |
| 10 | 安全在庫数 | 数値 | | アラート基準 |
| 11 | 発注者 | ユーザー | | |
| 12 | 使用期限 | 日付のみ | | |
| 13 | ロット番号 | 1行テキスト | | |
| 14 | 納品日 | 日付のみ | | |
| 15 | 受入検査日 | 日付のみ | | |
| 16 | COA入手日 | 日付のみ | 成績書入手日 |
| 17 | バーコード | 1行テキスト | | 将来用 |
| 18 | 備考 | 複数行テキスト | | |

### 選択肢列（複数選択可）

**保管場所**:
1. **「+ 列を追加」** → **「選択肢」**
2. 名前: `保管場所`
3. 選択肢を追加:
   - 204
   - 101
   - 未来
   - 7F
   - 動物舎
4. **「複数選択を許可する」** を**オン**
5. **「他の値を選択できるようにする」** を**オン**

**保管温度**:
1. **「+ 列を追加」** → **「選択肢」**
2. 名前: `保管温度`
3. 選択肢を追加:
   - 常温
   - 4℃
   - -30℃
   - -80℃
4. **「複数選択を許可する」** を**オン**
5. **「他の値を選択できるようにする」** を**オン**

---

## Step 3: 計算列の追加（オプション）

在庫金額を自動計算する場合：

1. **「リストの設定」** （⚙️ 歯車アイコン → リストの設定）
2. **「列の作成」** をクリック
3. 設定：
   - 名前: `在庫金額`
   - 種類: **「計算」**
   - 数式: `=[単価]*[現在在庫数]`
   - 戻り値の型: **「通貨」**
4. **「OK」** をクリック

---

## Step 4: 初期データ移行

1. リストビューで **「グリッドビューで編集」** をクリック
2. Excelからコピー＆ペーストでデータを入力
3. または **「エクスポート」** → **「CSV/Excelにエクスポート」** でテンプレートを取得し、データを入力後インポート

---

## 確認チェックリスト

- [ ] リスト「在庫管理」が作成できた
- [ ] 品名（タイトル）が必須になっている
- [ ] メーカー、メーカーコード、現在在庫数が必須になっている  
- [ ] 保管場所・保管温度が複数選択可能
- [ ] 在庫金額が自動計算される
